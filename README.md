# Support Ticket Simulations

A portfolio artifact showing how a support specialist works through real customer issues end-to-end.

## What this is

Three realistic support ticket scenarios—billing, permissions, and a feature sync issue—each presented with:

- the raw customer message (intentionally messy, as they are in real life)
- a clear, human support response
- internal notes covering root cause, what's being investigated, escalation rationale, and product gaps identified

This is not a product. It's a demonstration of support thinking and communication.

## How to run

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Purpose

Support simulations like these show how a specialist approaches ambiguous problems: reading between the lines of a customer message, drafting a response that is honest without over-committing, and capturing structured diagnostic notes for internal use. The goal is to make that thinking visible.
